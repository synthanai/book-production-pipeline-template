# archive/

**Purpose:** Archive for old preparation document versions

---

## What Goes Here

- Previous versions of master documents
- Old architecture specifications
- Superseded writing guides
- Previous blueprint versions
- Deprecated reference materials
- Old quality frameworks

## Organization

### By Version
```
archive/
├── v1.0-masters/
├── v2.0-guides/
└── v3.0-blueprints/
```

### By Date
```
archive/
├── 2025-11-01-Master-Story-Bible-v1.md
├── 2025-11-15-Guide-Writing-Excellence-old.md
└── 2025-12-01-Blueprint-Act-I-draft.md
```

## When to Archive

- Before major revisions to any prep document
- When consolidating multiple docs into one
- When prep doc structure changes significantly
- At major project milestones (e.g., before starting writing)

## Version Control

Include version number and date in archived filenames:
- `Master-Story-Bible-v1-2025-11-01.md`
- `Guide-Character-Portrayal-v2-2025-11-15.md`

## Notes

Preparation docs are foundational—keep all major versions. You may need to reference how decisions evolved over time.
